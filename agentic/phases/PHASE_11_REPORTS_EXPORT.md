# Phase 11: Reports and Export

## Goal
Let users view and export factual reports for themselves or their gynecologist.

## Scope
1. Add Reports screen.
2. Allow user to view:
   - symptom history
   - supplement adherence
   - meal history
   - nutrition gaps
   - weight trend
   - weekly AI summaries
3. Add date range filters.
4. Add export-ready data structure for gynecologist report.
5. If simple PDF export is feasible, add it.
6. If PDF export is not feasible in this phase, add a clean shareable text report.
7. Make sure report language is factual and non-diagnostic.
8. Include dates, pregnancy weeks, and logged user data.
9. Do not expose another user’s data.
10. Update documentation.

## Acceptance criteria
- Report screen exists.
- User can filter by date range.
- User can export or share a factual report.
- Report avoids diagnosis and medical conclusions.

## Required commit message
`Phase 11: Add reports and export`
